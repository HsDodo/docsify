## 👾第五章 树与二叉树

 <font size=4em color='#9ed048'>树的路径长度:</font> 是从树根到每个节点的路径长度的总和

 <font size=4em color='#c189'>满二叉树:</font>

​	对于编号为 i 的节点

​		①有双亲，则其双亲为看 是┗ i/2 ┛下取整

​		②若有左孩子，则左孩子为 2i ，若有右孩子，则右孩子为 2i+1

 <font size=4em color='#c189'>完全二叉树:</font>

​	对于编号为 i 的节点

​		①若 i<= ┗ i/2 ┛，则节点 i 为分支节点，否则为叶子节点 (__理解__ :编号最大的节点n，它的双亲节点则为最后一个分支节点)

 <font size=4em color='#c189'>满m叉树:</font>

​		①节点为 i 的第一个子女编号为__( i - 1)m+2__

 <font size=4em color='#c189'>线索二叉树:</font>

​		①二叉树是一种逻辑结构，但线索二叉树是加上线索后的链表结构，即它是二叉树在计算机内部的一种存储结构，所以是一种__物理结构__

​		②__后序线索树__的遍历仍需要栈的支持,因为当一个节点的右孩子不为空时( 右指针无法指向后继 )，因此通过指针可能无法遍历整棵树

​		

 <font size=4em color='#c189'>二叉排序树:</font>

​		①做题的时候注意别和 平衡二叉搜索树 弄混

​		

 <font size=4em color='#c189'>二叉平衡树:</font>

​		①N个节点的最大深度h,节点递推公式 N(0) = 0,N(1) = 1 ,N(2) = 2  ,N(h) = 1+N(h-1) + N(h-2) 

​		__理解__ :h为平衡二叉树的高度，N(h)为构造此高度的__最少__节点数(  <font size=4em color='8d4bbb'>注意这里是最少节点情况,也就是所有非叶子节点的平衡因子均为 1,两边一高一低</font> )  P182 -14

 <font size=4em color='#9ed048'>树和森林的遍历:</font>

​	①对应关系：

|    树    |   森林   |  二叉树  |
| :------: | :------: | :------: |
| 先根遍历 | 先序遍历 | 先序遍历 |
| 后根遍历 | 中序遍历 | 中序遍历 |

​	② 当要输出一个节点 n 到其祖先节点 m 的一条路径时，可以用后序遍历，因为到达 n 时，栈里面已经把 n 的祖先节点都存进去了

​	③二叉树线索化后，仍不能有效求解的问题是 __后序线索二叉树中求后序后继__ 

​	

 <font size=4em color='#ff7f50'>树的重要性质:</font>

​	① n个节点的树中有 n-1条边，"那么对于每棵树，其节点数比边数多1",例:森林中的节点数比边数多10，那么显然共有10棵树

​	②树转换为二叉树时，树的每个分支节点的所有子节点中的最右子节点无右孩子。
